## 計画対象のアプリ概要
ブラウザ上で動作する、2D迷路の自動生成および経路探索の可視化 Web アプリケーションを開発したい。

## システム要件
- クライアントサイドのみで完結させること
- 外部ライブラリやサーバーサイド処理には依存しないこと
- Vanilla JavaScript を使用すること
- PC、タブレット、スマートフォンに対応すること
- モバイルファーストなレスポンシブデザインを採用すること
- 画面サイズに応じて迷路を動的にスケーリングすること
- 選択された Difficulty などの設定値は localStorage に保存し、再訪時やリロード時に自動復元すること

## 必要な機能
1. Difficulty 設定
- グリッドサイズが選べるプルダウンを用意すること
- 例: Easy = 15x15, Normal = 25x25, Hard = 41x41
- 値は奇数サイズとすること
- 変更時に localStorage に保存すること

2. Generate Maze ボタン
- クリックで迷路を再生成すること
- 画面の表示状態を適切にリセットすること

3. Start Exploration ボタン
- クリックで探索アニメーションを開始すること
- 探索の進行をリアルタイムで可視化すること

4. 最短経路ハイライト
- ゴール到達後、スタートからゴールまでの最短経路を別アニメーションで強調表示すること

## アルゴリズム要件
- 迷路生成アルゴリズムは、完全迷路を生成でき、見栄えと実装性のバランスが良いものを選定すること
- 候補例: Recursive Backtracker など
- 経路探索アルゴリズムは、最短経路を保証し、視覚化との相性も良いものを選定すること
- 候補例: A* アルゴリズムなど
- 計画書では、採用アルゴリズムと採用理由も明記すること

## 技術方針
- 使用言語は HTML5、CSS3、JavaScript ES6+ とすること
- ロジックと描画を明確に分離すること
- 拡張性の高いオブジェクト指向設計を前提にすること
- 少なくとも以下の責務分割を前提に検討すること
  - MazeGenerator
  - MazeSolver
  - MazeRenderer
  - AppController
- 描画方式は canvas または DOM 描画のいずれかを比較し、採用方針を示すこと
- アニメーション実装では async/await を活用し、メインスレッドを極力ブロックしない方針を示すこと
